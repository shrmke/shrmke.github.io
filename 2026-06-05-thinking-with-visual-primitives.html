Thinking with Visual Primitives

把点和框放进多模态模型的思考过程，让模型不只是“看见”，还要在推理中明确“我指的是哪里”。

核心问题 怎么实现 训练流程 实验与边界 个人思考

从 Perception Gap 到 Reference Gap

一般 VLM 会把图像切成 patch，经过 ViT 得到视觉 token，再压缩或投影后和文本 token 一起送进 LLM。这篇论文没有推翻这个流程，底层仍然是类似 LLaVA 的 ViT + LLM 架构。

它真正想解决的是另一个问题：即使模型已经“看清楚”图像，纯语言 CoT 仍然很容易在复杂空间推理中指代漂移。模型说“左边那个物体”“上面的钥匙”“这条线”，但这些词没有稳定绑定到图像中的具体区域。作者把这称为 Reference Gap。

目标场景

场景为什么纯语言 CoT 容易漂移Visual primitives 的作用
密集计数对象多，容易漏数、重复数，后续“刚才那个”不稳定先用 box 锚定候选，再根据框集合计数
细粒度 VQA需要比较颜色、大小、材质、空间关系，多跳指代容易换目标每一步把被比较对象显式框出来
迷宫导航需要持续跟踪当前位置和分叉，语言很难表达连续路径用 point sequence 表达探索轨迹
路径追踪交叉线很多，必须沿同一条曲线持续追踪用一串 points 记录路径走向和终点
一句话：这篇论文不是提出一个全新的 VLM backbone，而是在标准 VLM 上提出一种 grounded multimodal reasoning framework，让隐式视觉注意力变成显式、可监督、可检查的中间状态。

把点和框变成“思考单位”

论文定义了两类 visual primitives：bounding box 和 point。前者适合对象定位和计数，后者适合轨迹、路径、拓扑推理等更抽象的空间引用。

输出格式

<ref>gold key</ref><box>[[566,110,649,165]]</box> <point>[[555,499],[597,472],[600,594],[544,682]]</point>

坐标被统一映射到 0-999 的归一化坐标系中，所以它不直接依赖原始图片分辨率。模型学习的是“图像区域”和“坐标表达”之间的对齐关系，而不是某个固定像素尺寸。

01
先拆任务
把问题拆成找目标、找候选、比较属性、排除干扰、得到答案。
02
再锚定对象
在关键步骤输出 box 或 point，明确当前推理引用的是哪个区域。
03
继续推理
后续步骤围绕已经显式化的目标、候选或路径状态展开。
04
最终回答
答案要和前面的视觉原语保持一致，例如框数量对应计数结果。

它和“步骤拆分”的关系

它确实是一种对思考过程的约束，但不是普通的“请一步步思考”。普通 CoT 只约束语言结构，这篇论文进一步约束中间视觉引用：模型必须把“我现在说的对象”落到图像坐标上。

方式中间状态能不能检查主要问题
纯语言 CoT“左边的物体”“上方的钥匙”很难自动检查指代模糊，容易漂移
Visual primitives CoT对象描述 + box / point可以用格式、坐标、答案一致性检查坐标可能标错，仍然需要监督和验证
工具化 crop / detector外部工具返回区域或候选 ID更可执行系统复杂度更高，依赖外部模块

训练：先学会指，再学会边指边想

这篇论文大量工作都在数据和训练流程上。核心目标是让模型不是偶尔输出坐标，而是在需要 grounding 的任务中稳定地把坐标放进推理链。

阶段数据来源 / 任务作用

Post-training pipeline

01
Specialized SFT
分别训练 box 专家和 point 专家，避免两种输出模式在小数据阶段互相干扰。
02
Specialized RL
用 GRPO 和 reward models 强化格式、质量、答案正确性。
03
Unified RFT
用专家模型生成更大规模数据，把 grounding 和 pointing 合成一个模型。
04
On-Policy Distillation
用两个专家教师模型蒸馏统一模型，缩小统一模型和专家之间的能力差距。

中间坐标怎么保证正确

论文并没有给出数学保证，而是用可验证数据和 reward 降低错误率。计数任务会检查 box 是否符合元数据、box 数量是否和答案一致；迷宫会检查路径是否穿墙；路径追踪会检查点序列是否贴合真实曲线、是否覆盖完整轨迹、终点是否正确。

关键判断：bbox / point 不是天然正确的真值锚点，而是一种可监督、可过滤、可惩罚的中间表示。它能降低漂移概率，但不能机制性消灭漂移。

Token efficiency

架构上，模型用 DeepSeek-ViT 和 DeepSeek-V4-Flash。论文举例，一个 756x756 图像先产生 2916 个 ViT patch tokens，经过 3x3 spatial token compression 后，进入 LLM 的视觉 token 约为 324，再通过 Compressed Sparse Attention 压到约 81 个视觉 KV entries。

CSA 到底是什么

CSA（Compressed Sparse Attention）不是本文提出的新 VLM 架构，而是 DeepSeek-V4-Flash 底座里的注意力 / KV cache 效率机制。它解决的是“视觉 token 和长上下文怎么算得省”，不是直接解决“推理怎么不漂移”。

步骤直觉理解关键点
Compress 把一小段 token 的 K/V 聚合成一个 compressed KV entry 不是简单平均，而是可学习的加权池化；默认压缩率可理解为每约 4 个 token 压成 1 个
Sparse 当前 query 先通过轻量 indexer 选择最相关的 compressed KV blocks 主注意力仍然存在，只是在筛选后的候选集合上做完整 attention
Trade-off 用近似 full attention 换显存和计算效率 可能漏掉重要信息，所以依赖低压缩率、overlap window、learned indexer 和训练适配降低风险
和标准注意力的关系：full attention 是“每次都看全部历史 K/V”；CSA 是“先用便宜的索引器找相关压缩块，再在这些块上做注意力”。所以它不是没有注意力，而是把注意力的候选集合缩小了。
贡献层级这篇论文做了什么怎么理解
VLM 架构仍是 ViT + LLM 的标准范式不是全新的 VLM backbone
视觉 token 效率3x3 压缩 + CSA KV cache 压缩主要继承 DeepSeek-V4-Flash 底座能力，用来支撑低视觉 token 成本
推理范式把 box / point 插入思考过程核心创新：visual primitives as reasoning units
训练范式SFT、RL、RFT、蒸馏围绕坐标推理链设计把中间视觉引用变成可学习目标

论文承认的局限

第一，受输入分辨率限制，细粒度场景中 visual primitives 仍可能不准。第二，目前需要显式 trigger 才会启用这种能力，未来希望模型自主决定何时使用。第三，point-based 拓扑推理仍然跨场景泛化有限。

个人判断：不是坐标万能，而是中间状态显式化

Takeaway 1

这篇论文最有价值的点不是“输出 bbox 就不会错”，而是把多模态推理中的视觉指代从隐式注意力变成显式中间状态。显式化以后，训练数据可以对齐，RL 可以惩罚，评测也可以检查。

Takeaway 2

bbox / point 更像一种约束化 CoT。它帮助模型把目标、证据和后续推理绑在一起，减少“我刚才说的那个对象到底是谁”的漂移。但如果坐标本身错了，后面的推理仍然可能建立在错误证据上。

Takeaway 3

这篇工作没有真正做到“输出坐标后再自动 crop 回图像局部重新看”。它主要还是在同一个自回归生成过程中学习坐标 token 和视觉 token 的对应关系。因此坐标是强上下文锚点，不是可执行视觉索引。

Takeaway 4

CSA 也不应被记成本文的新 VLM 架构。它是 DeepSeek-V4-Flash 底座里的 KV cache / sparse attention 机制：先压缩历史 K/V，再用 indexer 为当前 query 选相关压缩块。本文借它证明低视觉 token 成本下也能做强 grounded reasoning，但核心贡献仍然是 visual primitives 这个推理范式。

Caveat

如果把它扩展成系统，我会考虑加外部 verifier 或 crop 工具：模型先输出候选区域，系统裁剪放大并重新验证。那样 visual primitives 才从“文本里的坐标约束”进一步变成“可执行的视觉操作”。

可替代或互补的约束手段

手段作用和本文关系
候选对象 ID检测器先给 object list，模型只能引用 ID比自生成 bbox 更可控，但依赖检测器
Segmentation mask比 bbox 更精确，适合重叠和细长物体可以作为更细的 visual primitive
Crop / zoom-in 工具坐标区域被真正重新输入模型弥补本文“坐标不可执行”的弱点
程序化推理用 detect、filter、count、trace 等可执行步骤把中间过程从可检查推进到可执行
状态表维护已确认、已排除、当前位置、候选分支适合迷宫、UI 操作、长视频等多步任务
我的总结：这篇论文可以记成“给多模态 CoT 加视觉锚点”。它不是新的 VLM 架构，而是一个 grounded reasoning framework。真正的启发是：复杂 VLM 任务里，中间状态越显式、越可验证，模型越不容易在目标和证据之间漂移。
