One Model Is Enough

LLM 隐状态天然含有检索信息——加一个轻量 Projection Head，省掉整个 Embedding 模型

问题与动机 核心方法 实验结论 个人思考

传统 RAG Pipeline 的冗余问题

标准 RAG Agent 的检索流程需要两个独立模型，信息在中间被无谓地解码再重编码。

核心洞察：LLM 内部的隐状态已经是高质量的上下文表示，为什么要先解码成文字、再重新编码成向量？这个"隐状态 → 文字 → 向量"的路径本身就是信息的有损压缩。

核心方法：轻量 Projection Head

在 LLM 末尾接一个线性层，直接把最后一层的隐状态映射为检索向量，省掉整个独立 Embedding 模型。

信息路径对比

方案信息路径模型数量
传统 RAG隐状态 → 文字 → 向量（两次编解码）2 个
本文方案隐状态 → 向量（一步直达）1 个

实验结论

在 QReCC 对话检索基准上评测：

指标结果
Recall@10 保留率97%（相比两模型基线）
MRR@10 保留率97%
质量损失约 3%，换掉整个第二个模型
论文的定位不是"比传统方法更好"，而是"用 3% 的质量损失，换掉整个第二个模型"——这是一个工程权衡，而非检索质量的突破。

个人思考

值得质疑的角度
Embedding 模型本身就很轻量（100M～500M 参数），推理极快，延迟影响微乎其微。专用 Embedding 模型（BGE、E5）是专门为检索任务训练的，换成 LLM 隐状态的 Projection 反而损失了 3% 质量。"一个模型"的好处主要是概念上的简洁，工程上省不了多少。
学术价值
贡献更多是验证了一个洞察："LLM 隐状态天然含有检索信息"——这在理论上是有意义的。但对于已经有成熟 RAG pipeline 的团队，实际落地动力有限。
与 AttnRes 的共同精神
两篇论文都在说同一件事：不要浪费模型已经计算出来的信息。AttnRes 是"不丢弃早期层的隐状态"，本文是"不把隐状态先解码成文字再重新编码"。
